# Безопасность: чеклист middleware / auth / HTTP

Периодически пересматривать перед релизом и после крупных изменений в `internal/transport/http`.

## Автоматические проверки

| Команда | Назначение |
|---------|------------|
| `make security-scan` | `govulncheck` (уязвимости в зависимостях) + `gosec` (типичные ошибки в Go-коде) |
| `make lint` | golangci-lint |

## Middleware (`middleware/auth.go`, `server.go`)

- [ ] **JWT**: секрет только из env (`JWT_SECRET`), достаточная длина в проде; срок жизни токена осознан (`pkg/utils` JWT).
- [ ] **Authorization**: заголовок проверяется до бизнес-логики; нет обхода для «публичных» путей с чувствительными данными.
- [ ] **Контекст пользователя**: ключ контекста `"userID"` — при росте кода лучше заменить на неэкспортируемый тип `context.Key`, чтобы исключить коллизии с другими пакетами.
- [ ] **AdminOnly**: проверка `IsAdmin` по БД на каждый запрос — ок для строгости; при нагрузке можно кэшировать с осторожностью и коротким TTL.
- [ ] **CORS**: в проде задать `CORS_ALLOWED_ORIGINS` (список через запятую). Пустое значение = dev-режим `*` **без** `AllowCredentials` (соответствие спецификации Fetch). С куками/сессиями — только явные origin + credentials.
- [ ] **Таймауты**: `http.Server` Read/Write/Idle и `middleware.Timeout` согласованы с клиентами и долгими операциями.
- [ ] **Заголовки безопасности** (опционально): `X-Content-Type-Options: nosniff`, `X-Frame-Options`, `Referrer-Policy` — отдельный middleware при необходимости.

## Типичные OWASP (точечный обзор текущего кода)

| Тема | Статус / рекомендация |
|------|------------------------|
| **A01 Broken Access Control** | Защищённые маршруты под `Auth`; `/admin` + `AdminOnly`. Проверять IDOR на эндпоинтах с `{id}` (карта/тикет принадлежит пользователю). |
| **A02 Cryptographic Failures** | Пароли — хеш в utils; JWT HS256; секреты не в репозитории. |
| **A03 Injection** | SQL через sqlx с плейсхолдерами; не склеивать SQL строками. |
| **A05 Security Misconfiguration** | CORS см. выше; `AllowedHeaders` сужены относительно `*`. |
| **A07 Identification and Auth** | Логин/регистрация возвращают общие сообщения об ошибках — снижает перечисление аккаунтов (при необходимости ужесточить). |
| **A09 Logging** | Не логировать токены и пароли; wrapper для ошибок — не отдавать стек клиенту в проде. |
